路径加密

标签: 字符串

难度: Easy

假定一段路径记作字符串 path，其中以 "." 作为分隔符。现需将路径加密，加密方法为将 path 中的分隔符替换为空格 " "，请返回加密后的字符串。

示例 1：

输入：path = "a.aef.qerf.bb"

输出："a aef qerf bb"

限制：

0 <= path.length <= 10000

Submission

运行时间: 32 ms

内存: 13.2 MB

class Solution:
    def pathEncryption(self, path: str) -> str:
        res = []
        for char in path:
            if char == '.':
                res.append(' ')
            else:
                res.append(char)
        return ''.join(res)

Explain

该题解的核心思路是逐个字符遍历输入字符串 `path`。对于每个字符，检查其是否为点（`.`）。如果是点，则在结果列表 `res` 中添加一个空格字符（`' '`）；如果不是点，则将该字符直接添加到结果列表中。遍历完成后，使用 `join` 方法将列表中的字符合并成一个新的字符串，这样就完成了将所有的点替换为空格的任务。

时间复杂度: O(n)

空间复杂度: O(n)

class Solution:
    def pathEncryption(self, path: str) -> str:
        res = []  # 初始化一个空列表用于存放转换后的字符
        for char in path:  # 遍历输入字符串的每个字符
            if char == '.':  # 如果当前字符是点
                res.append(' ')  # 在列表中添加一个空格
            else:
                res.append(char)  # 否则，添加当前字符到列表
        return ''.join(res)  # 使用join方法将列表中的字符合并为一个字符串并返回

Explore

在Python中，字符串是不可变的对象，这意味着每次对字符串做出修改时，实际上是创建了一个新的字符串，而不是在原有字符串上直接修改。这种不断创建新字符串的过程，特别是在字符串较长时，会导致大量的内存分配和释放，从而降低程序的运行效率。相反，使用列表存储字符，可以在列表末尾高效地添加新元素，而不需要每次都创建新的列表。最终，使用 `join` 方法将列表中的字符合并为一个新的字符串，这样可以在保持高效的同时完成字符串的构建。

该算法可以正确处理空字符串的情况。因为算法是通过遍历输入的`path`字符串的每一个字符来进行操作的，如果`path`是空字符串，那么循环不会执行任何操作，最终`res`列表仍然为空。最后使用`join`方法将空列表转换为字符串时，结果自然是一个空字符串，符合预期输出。

当前算法在处理非常长的字符串时表现出良好的性能，因为列表的动态数组特性使得在末尾添加元素相对高效。然而，对于极端情况，即字符串长度达到或接近上限10000，该方法仍然需要遍历字符串的每个字符，并执行相应的条件判断和列表操作，可能会有一定的性能消耗。一个更高效的实现方法是使用 Python 内置的字符串 `replace` 方法，这个方法在底层已经进行了优化，可以直接将所有的`.`替换为`' '`，避免了额外的列表操作和条件判断，从而提高整体性能。
