删除链表中的节点

标签: 链表

难度: Medium

有一个单链表的 head，我们想删除它其中的一个节点 node。

给你一个需要删除的节点 node 。你将 无法访问 第一个节点  head。

链表的所有值都是 唯一的，并且保证给定的节点 node 不是链表中的最后一个节点。

删除给定的节点。注意，删除节点并不是指从内存中删除它。这里的意思是：

  • 给定节点的值不应该存在于链表中。
  • 链表中的节点数应该减少 1。
  • node 前面的所有值顺序相同。
  • node 后面的所有值顺序相同。

自定义测试：

  • 对于输入，你应该提供整个链表 head 和要给出的节点 node。node 不应该是链表的最后一个节点，而应该是链表中的一个实际节点。
  • 我们将构建链表，并将节点传递给你的函数。
  • 输出将是调用你函数后的整个链表。

示例 1：

输入：head = [4,5,1,9], node = 5
输出：[4,1,9]
解释：指定链表中值为 5 的第二个节点，那么在调用了你的函数之后，该链表应变为 4 -> 1 -> 9

示例 2：

输入：head = [4,5,1,9], node = 1
输出：[4,5,9]
解释：指定链表中值为 1 的第三个节点，那么在调用了你的函数之后，该链表应变为 4 -> 5 -> 9

提示：

  • 链表中节点的数目范围是 [2, 1000]
  • -1000 <= Node.val <= 1000
  • 链表中每个节点的值都是 唯一 的
  • 需要删除的节点 node 是 链表中的节点 ，且 不是末尾节点

Submission

运行时间: 40 ms

内存: 15.3 MB

# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def deleteNode(self, node):
        """
        :type node: ListNode
        :rtype: void Do not return anything, modify node in-place instead.
        """
        node.val = node.next.val
        node.next = node.next.next

Explain

这个题解的思路是直接将要删除节点的值替换为下一个节点的值，然后删除下一个节点。这样可以在无法访问头节点的情况下，通过修改要删除节点的值和next指针来达到删除节点的目的。

时间复杂度: O(1)

空间复杂度: O(1)

# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def deleteNode(self, node):
        """
        :type node: ListNode
        :rtype: void Do not return anything, modify node in-place instead.
        """
        # 将要删除节点的值替换为下一个节点的值
        node.val = node.next.val
        # 将要删除节点的next指针指向下下个节点，跳过下一个节点，相当于删除了下一个节点
        node.next = node.next.next

Explore

题解的基本假设是`node`不是链表中的最后一个节点。这个方法仅适用于给定的节点不是链表的尾节点的情况，因为我们需要访问`node.next`来复制其值和调整链接。如果`node`是最后一个节点，该方法将无法应用，因为没有下一个节点的值和链接可以使用来进行替换和删除。通常，调用这个函数的场景应当确保`node.next`确实存在。

是的，这种方法会暂时在链表中创建连续的相同值。当我们将`node.next`的值复制到`node`时，`node`和`node.next`将暂时持有相同的值。不过，通过将`node.next`指向`node.next.next`，我们实际上是在移除了原`node.next`节点，因此链表在操作完成后不会有重复的值存在。

在Python中，内存管理主要由垃圾回收器自动处理。当我们通过`node.next = node.next.next`移除对原`node.next`的引用时，如果没有其他引用指向该节点，该节点将变为垃圾回收器的回收目标。因此，在本题解中没有显式进行内存释放或资源管理，而是依赖于Python的垃圾回收机制来处理这些问题。

是的，此方法同样适用于只有两个节点的链表中删除第一个节点的情况。按照题解的方法，我们会将第二个节点的值复制到第一个节点，然后将第一个节点的`next`指向`None`（因为第二个节点之后没有其他节点）。这样，第一个节点就变成了链表中的唯一节点，第二个节点被移除，从而达到删除第一个节点的效果。
