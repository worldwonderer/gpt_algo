执行操作后的变量值

标签: 数组 字符串 模拟

难度: Easy

存在一种仅支持 4 种操作和 1 个变量 X 的编程语言：

  • ++X 和 X++ 使变量 X 的值 加 1
  • --X 和 X-- 使变量 X 的值 减 1

最初，X 的值是 0

给你一个字符串数组 operations ，这是由操作组成的一个列表，返回执行所有操作后， X 的 最终值 。

示例 1：

输入：operations = ["--X","X++","X++"]
输出：1
解释：操作按下述步骤执行：
最初，X = 0
--X：X 减 1 ，X =  0 - 1 = -1
X++：X 加 1 ，X = -1 + 1 =  0
X++：X 加 1 ，X =  0 + 1 =  1

示例 2：

输入：operations = ["++X","++X","X++"]
输出：3
解释：操作按下述步骤执行： 
最初，X = 0
++X：X 加 1 ，X = 0 + 1 = 1
++X：X 加 1 ，X = 1 + 1 = 2
X++：X 加 1 ，X = 2 + 1 = 3

示例 3：

输入：operations = ["X++","++X","--X","X--"]
输出：0
解释：操作按下述步骤执行：
最初，X = 0
X++：X 加 1 ，X = 0 + 1 = 1
++X：X 加 1 ，X = 1 + 1 = 2
--X：X 减 1 ，X = 2 - 1 = 1
X--：X 减 1 ，X = 1 - 1 = 0

提示：

  • 1 <= operations.length <= 100
  • operations[i] 将会是 "++X"、"X++"、"--X" 或 "X--"

Submission

运行时间: 18 ms

内存: 16.1 MB

from typing import List

class Solution:
    def finalValueAfterOperations(self, operations: List[str]) -> int:
        X = 0  # 初始化变量 X 的值为 0
        
        for operation in operations:
            if operation == "++X" or operation == "X++":
                X += 1  # 执行加 1 操作
            elif operation == "--X" or operation == "X--":
                X -= 1  # 执行减 1 操作
        
        return X  # 返回最终值

solution = Solution()
operations = ["--X", "X++", "X++"]
result = solution.finalValueAfterOperations(operations)
print(result)  # 输出: 1

Explain

该题解通过迭代输入的操作列表来更新变量X的值。对于每个操作，根据其具体是增加还是减少操作，决定是对X进行加1或减1的操作。最终返回经过所有操作后的X的值。

时间复杂度: O(n)

空间复杂度: O(1)

Explore

在Python中，使用`for operation in operations:`循环是因为这种方式直接、清晰并且效率高。这种循环直接遍历操作列表，易于理解和实现。使用while循环亦可，但需要手动管理索引，增加了代码复杂性。递归虽然理论上可行，但在处理大量操作时可能导致堆栈溢出，并且递归在这种简单的迭代任务中通常没有性能优势，反而更为复杂和低效。

在实际应用中，对操作字符串进行错误或格式检查是非常有必要的，以确保程序的健壮性和错误处理能力。如果操作数组包含非法字符串，应该在执行操作前进行验证。可以抛出异常或返回错误信息，以通知调用者输入包含不合法的操作。这样可以避免程序在运行时因未预期的输入而产生错误或异常行为。

将变量`X`作为一个参数传入函数确实是一个灵活的设计选择，它允许函数调用者指定不同的初始值。这种设计可以使函数更加通用和灵活。在当前的实现中，`X`被初始化为0，目的是为了简化问题的解决方案。如果需要支持更广泛的用例，可以修改函数定义，添加一个额外的参数来允许传入初始值。
